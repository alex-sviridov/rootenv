# Contmgr

**Stack:** Go ‚Äî single long-running process. Docker containerized.

## Responsibilities
- Polls PocketBase every `CONTMGR_POLL_INTERVAL` (default 5s) for assets in `provisioning` state
- Spins up Docker containers for each asset, injects SSH public key, records connection details
- Generates Ed25519 SSH keypairs; encrypts private key with AES-256-GCM using `keys.secret` as key material
- Writes encrypted private key to `keys.key_encrypted` in PocketBase
- Polls for pending `decommission` commands and removes containers accordingly
- Authenticates with PocketBase as `svc_contmgr@contmgr.local` (svc_role="contmgr")
- Stateless: all state lives in PocketBase; multiple instances safe to run in parallel

## Key Config Env Vars
| Var | Default | Description |
|---|---|---|
| `CONTMGR_BACKEND_URL` | ‚Äî | PocketBase base URL |
| `CONTMGR_BACKEND_USERNAME` | ‚Äî | svc_contmgr email |
| `CONTMGR_BACKEND_PASSWORD` | ‚Äî | password |
| `CONTMGR_HOST_IP` | ‚Äî | IP reachable from sibling containers (e.g. `172.17.0.1`) |
| `CONTMGR_POLL_INTERVAL` | `5s` | Poll cadence |
| `CONTMGR_DOCKER_HOST` | `unix:///var/run/docker.sock` | Docker socket |

## Encryption
- AES-256-GCM; key = SHA-256(`keys.secret`); stored as `base64(12-byte-nonce || ciphertext)`
- `keys.secret` is a 32-char alphanumeric string auto-generated by PocketBase
- Relay uses the same scheme to decrypt at connection time ‚Äî the two must stay in sync

## Port Assignment
`net.Listen("tcp", ":0")` ‚Üí capture port ‚Üí close listener ‚Üí publish to container. Race window accepted; containers are ephemeral and internal.

## SSH Key Injection
Docker exec after container start: writes authorized_keys file inside container at `/conf.d/authorized_keys/<ssh_user>`.

## Concurrency
`errgroup` for concurrent provision/decommission; errors per-asset are logged but do not abort the poll cycle.

## Constraints
- No TLS or SSH host key pinning ‚Äî `InsecureIgnoreHostKey` is correct for ephemeral internal containers
- No image pre-pull ‚Äî assumes image is available on host
- No multi-host Docker ‚Äî single host via docker.sock
- Only provisions and decommissions ‚Äî start/stop/restart commands are not its responsibility
- Does NOT expose any ports or HTTP endpoints
